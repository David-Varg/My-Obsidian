Un puntero es una variable que no guarda un valor directamente, sino la *dirección* de Memoria donde está ese valor.

Básicamente, dice en que lugar de la memoria se guardó tus datos, es "la dirección de tu casa". Te permite manipular directamente lo que está en [[02. Stack vs. Heap (Control de Memoria Profunda)|Stack]] o en [[02. Stack vs. Heap (Control de Memoria Profunda)|Heap]].

**Por ejemplo:**
Si tenemos algo como esto:
```

STACK
p → 0x200 (puntero)
x → 10 (variable normal)
```

Y hacemos algo como esto:
```C

int x = 10;
int *p = &x;
```

- *p* guarda la dirección de *x (0x200)*, no del 10
- Si cambias `*p = 20;` entonces *x* ahora es 20, porque el puntero modificó directamente el valor al que apuntaba.

### **Relación con Stack y Heap**
- **Stack**: Los punteros pueden apuntar a variables locales. → *cuidado*: si la función termina, el stack desaparece y el puntero queda colgado (**dangling pointer**)
- **Heap**: Los punteros pueden apuntar a objetos dinámicos. → permite manipular estructuras que viven mientras haya referencias.

### **¿Por qué es importante?**
Cada vez que haces:
- *Listas enlazadas* → Cada nodo tiene un puntero al siguiente. [[06. Listas Enlazadas|Ver más...]]
- *Árboles* → Cada nodo tiene punteros a hijos. [[11. Árboles|Ver más...]]
- *Grafos* → Punteros a los vecinos. [[09. Grafos|Ver más...]]
- *Tablas Hash* → Punteros a buckets. [[10. Tablas Hash|Ver más...]]

Todo es manipulación de direcciones.

---

#### **Cosas a tener en cuenta**
En Lenguajes de *Bajo Nivel* los punteros los puedes controlar manualmente, pero en Lenguajes de *Alto Nivel* tú no puedes manipular los punteros directamente sino que el lenguaje los maneja por ti.

Aquí nuevamente están los mismos ejemplos en donde lenguajes como *C* o *C++* se puede manipular los punteros y otros como *Javascript*, *Python*, *Java* o *C#* no.

- **¿Por qué estudiar punteros?**
Porque las estructuras de datos funcionan conceptualmente con punteros.

Ejemplo de Lista Enlazada:
```

[nodo1 | next] → [nodo2 | next] → [nodo3 | next]
```

Cada *next* es literalmente un puntero a otro nodo.

Aunque Python lo esconda, la lógica es la misma.

- **Ejemplo de Dangling Pointer**
Esto solo pasa cuando controlas memoria manualmente.

*Ejemplo en C*:
```C

int* crear() {
    int x = 5;
    return &x;
}
```

*X* está en el **[[02. Stack vs. Heap (Control de Memoria Profunda)|Stack]]** de la función. Cuando la función termina el [[02. Stack vs. Heap (Control de Memoria Profunda)|Stack]] desaparece pero, como devolviste su dirección (Puntero) entonces obtienes esto:
```

puntero → 0x100 (dirección vacía)
```

Eso es un Dangling Pointer.

---

#### Resumen rápido:
```

variables → referencias (stack)
objetos → memoria (heap)
```

Hay dos formas de operar con punteros:
1. Modificar el Objeto (Mutabilidad)
```python

a = [1,2,3]
b = a
a.append(4)
```

Esto no cambia el puntero:
```

a---|
    |---- Mismo Objeto
b---|
```

Entonces ambos ven el cambio.

2. Reasignar la variable (Reasignación)
```python

a = [1,2,3]
b = a

a = [9,9,9]
```

La variable apunta a un objeto nuevo:
```

a----> Objeto Nuevo

b----> Objeto Viejo
```
