No es una estructura de Datos, sino que es la forma para entender cómo viven los programas en memoria.

### **Stack**
El stack es... Básicamente:
> Memoria automática, reservada para datos temporales, está ordenada.

- Se organiza en formato *LIFO (Last In-First Out)*.
- Es muy rápida por su acceso lineal.
- Es limitada y fija en tamaño (Depende del lenguaje y del SO).
- Se gestiona automáticamente. (Depende del lenguaje)
- Es temporal.
- Contenido: Variables locales y tipos de datos simples (o referencias).

Al acabarse la ejecución de un Frame (Ya sea el principal o el de una función) se destruye el Stack.

Un ejemplo es que, cada vez que llamas a una función:
1. Se queda esperando el resto del programa.
2. Se crea un "frame" (Un espacio temporal de memoria)
3. Se guardan variables locales (Variables que solo existen dentro de esa función)
4. Cuando termina la función, se elimina todo lo que fuese creado dentro de ella.
Como una torre que se arma y se desarma sola.

>[!important] **Stack Overflow**
>Cuando te excedes y llegas al limite de *Stacks* en Memoria ocurre un desbordamiento en memoria causado por Stack, también llamado *Stack Overflow*.
>Esto ocurre porque los *Stacks* tienen un  tamaño fijo reservado en Memoria y no se puede exceder ese limite.

También existen las **Call Stacks** que son básicamente funciones que llaman a otras funciones dentro de ellas. Al hacer esto la función principal se congela y hasta que la nueva función no termine su proceso, seguirá congelada.

###### Ejemplo:
```python

def saludar():
    nombre = obtener_nombre()
    console.log(nombre)

def obtener_nombre():
    username = input("Nombre: ")
    return username

saludar()
```

- Primero entra *saludar()*.
- Luego entra *obtener_nombre()* encima.
- Cuando *obtener_nombre()* termina, desaparece y retorna simplemente **username**
- Luego *saludar()* obtiene el **username** y termina.

Eso es el **Stack**, cuando la función termina esa memoria se limpia automáticamente.

>No confundir con el [[07. Pilas (Stacks)|Stack]] de Estructura.

### **Heap**
El Heap es...:
>Memoria dinámica donde viven objetos creados en tiempo de ejecución.

- Más grande.
- Más lento.
- No se libera automáticamente (Depende del lenguaje).
- Vive hasta que la liberes o el Garbage Collector actúe.

Normalmente solo se guardan Tipos de Datos Complejos (Listas, diccionarios, arboles, etc.), *no* Primitivos.

Ejemplo:
```python

lista = [1, 2, 3]
```

```

STACK
lista → dirección 0xA13F

HEAP
0xA13F → [1, 2, 3]
```

O sea:
- El **Stack** es donde se guardan las *referencias*.
- El **Heap** es donde se guarda el *objeto real*.

Punto a tener en cuenta es que, como ya dije anteriormente, solo se guardan datos complejos (objetos, por ejemplo). Si tienes:
```js

let a = 5

const suma = (x) => x += 5

suma(a)
```

Aquí normalmente se creería que *a* y *x* como en el ejemplo anterior, hará referencia a **5**. Pero no, el **5** al ser un dato del tipo primitivo solo se guarda en el **Stack**. Aquí no está presente el **Heap**:
```

STACK
a → dirección 0xB42C
x → dirección 1xA81D
```

>[!important] **Aclaración:**
>En lenguajes como *Python* (Por ejemplo) todo es un **Objeto**, por eso este ejemplo no sirve allí ya que se guarda directamente en el Heap, pero en otros esto no ocurre.

El **Heap** es eliminado automáticamente en Lenguajes de *Alto Nivel (Python, JS)* pero en Lenguajes de *Bajo Nivel (C, Assembler)* tiene que ser eliminado manualmente.

Hay algo llamado **Memory Leak**. En lenguajes de *Alto Nivel* gracias al *Garbage Collector (GC)* no te va a suceder este error y como Máximo se pondrá lento el programa, pero en lenguajes de *Bajo Nivel* como C, C++ o Assembly si puede ocurrir.

Ocurre cuando se elimina el **Stack** o el [[04. Punteros|Puntero]] que se relacionaban con el **Heap**, y no se eliminó previamente. La información del **Heap** queda vagando por la memoria como un zombie y es imposible de que se pueda limpiar ese dato.

Si por ejemplo creas un [[05. Arreglos|Array]] adentro de un Bucle, al terminar la primera vuelta se va a **eliminar el Stack** y si el bucle sigue y sigue, aunque sea por tan solo unos segundos entonces puede salir el **Pantallazo Azul (En SO Modernos no ocurre normalmente)**, **Crashearse el Computador** o volverse **Extremadamente Lento** y la *RAM* usada no se podrá liberar a menos de que se reinicie el computador y además, si el problema llegó a los *servidores* entonces se tendrá que reiniciar el servidor en donde está alojado la *Aplicación/Web*.

>No confundir con [[12. Árboles → Heaps|Heaps]] de Estructura.
