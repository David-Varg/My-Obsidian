### **¿Que son?**
Las listas enlazadas a primera vista parecen [[05. Arreglos|Arrays]] convencionales, solo que estas tienen una característica: *No son contiguas*.

Los Arrays son listas contiguas pero estas no lo son, o por lo menos no están ordenadas así en Memoria, están constituidas por 3 cosas:
- *El head*
- *El Nodo*
- *El Valor*

El *head* es el Nodo que está al inicio, el que encabeza la Lista. El *Nodo* es el apartado en Memoria en donde está guardado *Valor* al cuál nos vamos a conectar. Con esto, se consiguen 3 tipos de **Linked Lists**:
1. Singly Linked List
2. Double Linked List
3. Circular Linked List

### **¿Por qué usarlo?**
A diferencia de los [[05. Arreglos|Arreglos]], estos no pueden acceder a un dato de forma directa ($O(1)$) sino que su composición es secuencial ($O(n)$) porque no comparten el mismo espacio en Memoria.

A cambio tienen una flexibilidad que los *Arreglos* (Que son rigidos) no poseen; si se quiere introducir un dato en medio o al inicio de toda la lista (una Lista tipo *LIFO* por ejemplo) simplemente corta la conexión entre los dos *Nodos* y se coloca en medio ($O(1)$).

Por lo que usa una *Linked List* si:
- No sabes cuántos elementos vas a tener y la lista cambiará mucho.
- Vas a insertar o eliminar frecuentemente (especialmente al inicio).
- Quieres implementar otras estructuras como [[07. Pilas (Stacks)|Pilas (Stacks)]] o [[08. Colas (Queues)|Colas (Queues)]] de forma eficiente.

### **Tipos de Linked List**
Como se dijo anteriormente, hay 3 tipos de Listas Enlazadas.
#### **Singly Linked List**
Siguen el siguiente formato:
![[Linked List Normal]]
En donde el Cuadrado es el *Nodo* y la información está adentro del Cuadrado. El último nodo está vinculado a un valor *Nulo (Null)* para indicar de que ahí acaba la lista.

Usaremos el *Paradigma POO* ([[01. Paradigmas de Programación]]) y *Javascript* en los ejemplos (Pero también se puede hacer con Objetos/Diccionarios normales).

Primero, se creará la clase *Nodo*:
```js

class newNodo {
    constructor(value){
        this.value = value
        this.next = null
    }
}
```

El *next* lo iniciamos en **null** debido a que este es solo el Esquema y lo usaremos después.

El siguiente paso es, ahora si, crear la clase de la *Linked List* con sus respectivos metodos:
```js

class singlyLinkedList {
    constructor() {
        this.head = null
    }
}
```

¿Por qué lo iniciamos así? Simple: porque aún no está creada la Lista, entonces aún el head no apunta hacia ningún lado.
