# 📖 Glosario de Ingeniería de Software

> [!abstract] Objetivo
> Definiciones breves y precisas de términos recurrentes. Si el término requiere más de 3 párrafos, debe tener su propia nota vinculada.

---

## 🅰️ A
- **Abstracción**: Proceso de ocultar detalles de implementación para enfocarse en lo esencial. Ver: [[07. Abstracción]].
- **Algoritmo**: Secuencia finita de instrucciones bien definidas para resolver un problema específico. Ver: [[08. Pensamiento Algorítmico]].
- **API**: Interfaz que define cómo interactúan componentes de software entre sí. Ver: [[07. APIs]].

## 🅱️ B
- **Backend**: Capa del sistema que maneja lógica de negocio, datos y procesamiento. Ver: [[04_MOC BackEnd]].
- **Big-O**: Notación asintótica que describe el comportamiento de un algoritmo en el peor caso. Ver: [[15. Complejidad Algorítmica]].
- **Binario**: Sistema numérico base-2 que representa datos en computadoras. Ver: [[02. Binario y Lógica]].

## 🅲️ C
- **Cache**: Mecanismo de almacenamiento temporal para acceso rápido a datos frecuentes. Ver: [[14. Cache (HTTP Caching)]].
- **Clean Code**: Prácticas para escribir código legible, mantenible y expresivo. Ver: [[07. Clean Code]].
- **Cliente-Servidor**: Arquitectura donde clientes solicitan servicios a servidores centralizados. Ver: [[03. Arquitectura Cliente-Servidor]].
- **Chain of Thought (CoT)**: Técnica de prompt donde se pide a la IA que "piense paso a paso" para mejorar la precisión. Ver: [[04. Chain of Thought]].
- **Comunicación Técnica**: Habilidad de explicar conceptos complejos de forma simple a diferentes audiencias. Ver: [[01. Comunicación Técnica]].

## 🅳️ D
- **DOM**: Árbol de objetos que representa la estructura de una página web. Ver: [[09. DOM]].
- **DRY**: Principio "Don't Repeat Yourself" para evitar duplicación de lógica. Ver: [[10. DRY-KISS-YAGNI]].
- **Debugging**: Proceso sistemático de identificar y corregir errores en el código. Ver: [[15. Manejo de Errores (Debugging)]].

## 🅴️ E
- **Escalabilidad**: Capacidad de un sistema para manejar mayor carga manteniendo rendimiento. Ver: [[09. Escalabilidad básica]].
- **Estado**: Datos que pueden cambiar durante la ejecución de un programa.
- **Escuchas de Eventos (EventListeners)**: Funciones que esperan a que el usuario haga algo (tecla, clic) para reaccionar. Ver: [[10. Programación Asíncrona]].
- **Endianness**: Orden en que se organizan los bytes de un dato en la memoria (Big-endian vs Little-endian). Ver: [[09. Endianness]].
- **Ética Profesional**: Conjunto de normas morales que rigen la conducta en el desarrollo de software. Ver: [[10. Ética y Responsabilidad Profesional (Ampliado)]].
- **Ética en IA**: Conjunto de principios para evitar sesgos y asegurar el uso responsable de modelos de lenguaje. Ver: [[08. Ética y Sesgos en IA]].

## 🅵️ F
- **Frontend**: Capa de interfaz de usuario que interactúa directamente con el usuario. Ver: [[03_MOC Web]].
- **Funciones**: Bloques reutilizables de código que encapsulan lógica específica. Ver: [[06. Funciones y Modularización]].
- **Feedback**: Información de retorno sobre un proceso o código para mejorarlo. Ver: [[07. Feedback y Code Review]].

## 🅶️ G
- **Git**: Sistema distribuido de control de versiones para rastrear cambios. Ver: [[05. Git y Control de Versiones]].
- **Grados**: Número de conexiones directas de un nodo en estructuras de grafos.
- **Grafos**: Conjunto de nodos (vértices) que almacenan datos, conectados por enlaces (aristas) que representan relaciones entre ellos. Ver: [[09. Grafos]]

## 🅷️ H
- **Hash**: Función que mapea datos de tamaño variable a valores de tamaño fijo. Ver: [[10. Tablas Hash]].
- **Heap**: Estructura de memoria dinámica para almacenamiento en tiempo de ejecución. Ver: [[02. Stack vs. Heap (Control de Memoria Profunda)]].
- **HTTP**: Protocolo de aplicación para transferencia de hipertexto en la web. Ver: [[04. HTTP]].
- **Habilidades Blandas (Soft Skills)**: Competencias interpersonales como comunicación y liderazgo, críticas para trabajar en equipo. Ver: [[12_MOC Soft Skills]]

## 🅸️ I
- **Inmutabilidad**: Propiedad de datos que no pueden modificarse tras su creación. Ver: [[02. Inmutabilidad]].
- **Índices**: Estructuras que aceleran la búsqueda en bases de datos. Ver: [[04. Índices]].
- **IEEE 754**: Estándar técnico para la representación de números de punto flotante. Ver: [[08. IEEE 754]].

## 🅹️ J
- **JWT**: Token web JSON auto-contenido para transmisión segura de claims. Ver: [[04. JWT]].

## 🅺️ K
- **KISS**: Principio "Keep It Simple, Stupid" para diseños simples y directos. Ver: [[10. DRY-KISS-YAGNI]].

## 🅻️ L
- **Lógica Proposicional**: Sistema formal para razonar con afirmaciones verdaderas/falsas. Ver: [[02. Lógica Proposicional]].
- **LLM**: Modelo de Lenguaje de Gran Escala (como Gemini o GPT) entrenado con billones de palabras. Ver: [[01. Fundamentos del Modelo de Lenguaje]].

## 🅼️ M
- **Middleware**: Software que conecta componentes de aplicación en el flujo de procesamiento. Ver: [[02. Middleware]].
- **MVC**: Patrón arquitectónico que separa Modelo, Vista y Controlador. Ver: [[07. MVC]].
- **Microservicios**: Arquitectura que compone aplicaciones de servicios independientes. Ver: [[09. Monolito vs. Microservicios]].
- **Matriz**: Estructura de datos bidimensional (un array de arrays). Por ejemplo, en Tetris, representa el tablero. Ver: [[01. Matemática Discreta]]
- **Mentalidad de Crecimiento**: Creencia de que las habilidades se desarrollan con esfuerzo y práctica. Ver: [[09. Mentalidad de Crecimiento Profesional]].
- Mapear: Proceso de transformar un conjunto de datos en otro, o establecer una correspondencia entre dos estructuras. Ver también: [[03. Variables y Tipos de Datos]], [[05. ORM]].

## 🅽️ N
- **NoSQL**: Bases de datos no relacionales diseñadas para datos no estructurados. Ver: [[06. NoSQL]].
- **Normalización**: Proceso para eliminar redundancias en bases de datos relacionales. Ver: [[03. Normalización]].

## 🅾️ O
- **ORM**: Herramienta que mapea objetos de código a tablas de bases de datos. Ver: [[05. ORM]].

## 🅿️ P
- **Paradigma**: Estilo fundamental de organización y estructuración de programas. Ver: [[01. Paradigmas de Programación]].
- **Polimorfismo**: Capacidad de objetos de diferentes tipos responder al mismo mensaje.
- **Prompt Engineering**: El arte de diseñar instrucciones precisas para obtener los mejores resultados de una IA. Ver: [[02. Prompt Engineering básico]].

## 🆀️ Q
- **Query**: Consulta estructurada para recuperar o modificar datos en sistemas.

## 🆁️ R
- **Recursión**: Técnica donde una función se invoca a sí misma para resolver subproblemas. Ver: [[22. Recursión]].
- **REST**: Estilo arquitectónico para diseño de APIs web stateless. Ver: [[06. REST]].
- **Refactorización**: Reestructuración de código sin cambiar su comportamiento externo. Ver: [[08. Refactorización]].
- **RAG**: Técnica que conecta una IA a tus propios documentos para que responda basada en información real. Ver: [[05. RAG (Retrieval-Augmented Generación)]].

## 🅢️ S
- **Scrum**: Marco ágil para desarrollo iterativo con roles y ceremonias definidas. Ver: [[12. Metodologías (Scrum)]].
- **SQL**: Lenguaje declarativo para gestionar bases de datos relacionales. Ver: [[02. SQL]].
- **SOLID**: Conjunto de cinco principios para diseño orientado a objetos. Ver: [[09. Principios SOLID]].
- **Stack vs Heap**: Dos formas diferentes en que la computadora organiza la memoria. El Stack es rápido y ordenado; el Heap es grande y flexible. Ver: [[02. Stack vs. Heap (Control de Memoria Profunda)]].
- **Stakeholders**: Personas interesadas o afectadas por un proyecto de software. Ver: [[02. Ingeniería de Requerimientos]].

## 🅣️ T
- **Transacciones**: Unidad de trabajo que garantiza propiedades ACID en bases de datos. Ver: [[05. Transacciones (ACID)]].

## 🆄️ U
- **UML**: Lenguaje de modelado unificado para visualizar sistemas de software. Ver: [[03. UML y Modelado]].

## 🆅️ V
- **Variables**: Contenedores con nombre que almacenan valores mutables. Ver: [[03. Variables y Tipos de Datos]].

## 🆆️ W
- **WebSocket**: Protocolo de comunicación bidireccional persistente sobre TCP. Ver: [[12. WebSockets]].

## 🆇️ X
- **XSS**: Vulnerabilidad que permite inyectar scripts maliciosos en páginas web. Ver: [[04. XSS]].

## 🆈️ Y
- **YAGNI**: Principio "You Ain't Gonna Need It" contra sobreingeniería. Ver: [[10. DRY-KISS-YAGNI]].

## 🆉️ Z
- **Zero Trust**: Modelo de seguridad que nunca confía, siempre verifica.
